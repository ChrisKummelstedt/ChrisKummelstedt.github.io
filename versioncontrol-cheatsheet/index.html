VersionControl.Cheatsheet

Christopher Kummelstedt

Check if you have git

$ git --version Check version

Configure git

git config --global user.name "your name" Configure username

git config --global user.email "your email" Configure email

git config --global core.autocrlf input Configure ??

git config --global core.safecrlf true Configure ??

Setting up a git repository

mkdir gitDir Create a new directory

cd gitDirChange into it

echo "Git is Awesome" > gitText Create a file

git init Changes current directory to git repo

Git repo - creates a hidden directory that tracks changes to files in the directory.

NB! You should run git init inside the directory where the files for the project you're placing under version control are.

If you do it somewhere else you can use (rm -rf .git = remove recursively, without prompting) to remove it.

git add gitText Adds file to staging area

git rm gitText Removes a file from staging area

git commit -m "First commit" Commit all files in staging area

Example of success:

$ git commit -m "First commit" [master (root-commit) a1833e4] First commit 1 file changed, 1 insertion(+)
create mode 100644 gitText

git status Gives overview of what happened since last time

git log shows a logg

git checkout a1833e4ef4a1b change back to commit with hash

git checkout master go back to current master

Connecting computer with Github repo

Add a SSH key: https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/

Create repo using interface

git remote -v Check if you have any remotes (connections between a local repo and a github repo)

git remote add origin NEWREPOURL Add remote

Git push -u origin master Push master to origin and make it the default action of git push (-u)

git remote rm origin Removing a remote
