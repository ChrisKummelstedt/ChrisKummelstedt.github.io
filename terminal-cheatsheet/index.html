CommandLine.Cheatsheet

Christopher Kummelstedt

This is a cheatsheet for basic command line stuff learned in week one of Makers Academy's pre-course.

Terminal terminology

|
Pipe/Redirect stream
stdin
Standard stream in
stdout
Standard stream out
stderr
Standard error

General Terminal commands

ls Lists directory content

ls –l –A Using two switches (L for long format and A for all files hidden files are part of it)

cd ~/… Changes folder

whoami Identifies user

wc Counts lines, words and characters

date Prints date

pwd Print working directory (shows path to current folder)

touch NAME Create a new file with NAME

mkdir NAME Create a new folder with NAME

rmdir NAME Delete a folder with NAME

rm NAME delete a file with NAME use with caution

rm –r DIRECTORY -r (recursively) remove DIRECTORY

head -n 20 file top 20 rows

rm –i -i (interactively), i.e. it will ask for each file

rm –f -f (forced) even write protected files will burn use with caution

cp OLD NEW create copy OLD file as NEW file

mv NAME ../ move file NAME to folder

cat concatenate i.e. combine

/private/var/log/system.log what happened?

man ls brings up the manual for ls

Combining terminal commands

find . -name "*.txt" –print

find all files, starting with the current directory, with any name that ends in .txt and print it to the screen

grep WORD *txt

Search all *txt files and print all that contain WORD

|

Redirect stream

find . -name "*.txt" –print | grep WORD

Find all files, starting with the current directory, with any name that ends that .txt and redirect this stream to grep, grep should in turn display any pwd/file that contains word

find . -name "*.txt" –print > list.txt

Find all files, starting with the current directory, with any name that ends that .txt and create a new file containing this stream, called list.txt

find ~ -name "*.txt" -print | grep README | wc –l

Find all files, starting with the current directory, with any name that ends that .txt, then check which of those contain the word README, then display how many lines (giving us how many files)

Permissions

Types: Read, Write, Execute Class: User, Group, Others

ls –l

You can view this using long format

This presents the info on permission in the following way

USER (RWX) – GROUP (RWX) – OTHERS (RWX)

Changing permissions:

  • chmod u+w readme.txt gives reading permissions to users
  • whoami gives user

Stuff to do with interpreters, paths, env and processes

Which ruby Gives path to ruby interpreter

#! Shebang; tells computer what interpreter to use

#!/usr/bin/env ruby Loads the right interpreter for ruby

echo $HOME Gives my home directory

echo $PATH Gives a colon separated list of places the computer will look for programs.

*NB! All programs that don’t require a shebang, or alternative with similar function, live in a folder specified by path *

Env Show all key-value pairs in environment

export KEY=Value Set new pair in current terminal session

echo $KEY Echo’s value

export PATH=$PATH:/Users/fluff Add another path directory

echo "export SEASON=winter" >> ~/.bash_profile if you want to keep KV pair in your profile (keep permanently)

NB! >> means append.

ps processes currently doing something in terminal

ps x all processes (in all windows)

ps x | grep BASH all processes (in all windows) involving BASH

ps x | grep BASH | wc -l makes a list of all processes (in all windows) involving BASH and then counts how many lines are on that list (i.e. how many such processes there are)

Basic VIM

vi myFile create file i start inputting before cursor o open a new line after the current one dd delete current line and dozens of others

how to edit in VIM

  1. i (input)
  2. insert stuff
  3. press esc
  4. :w (write)
  5. :q (quit)
  6. :q! (quit without saving)
