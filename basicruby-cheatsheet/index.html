1.2 Float that integer

If you want a value in float form (i.e. not an integer) you can manipulate it with a float:

  • X == 2
  • X / 1.0
  • return X

Will return 1.0 (rather than 1)

1.5 Subing parts of a string

  • MyArray = MyString.split(" ")

Turning a string into an array by splitting it at " " (spaces)

  • MyArray.join(" ")

Turning an array into string by adding " " (spaces) between elements.

The correct answer uses string.gsub('old', 'new'). As I forgot about this useful tool I did this strange thing - first I had some problems changing the array elements to happy/Happy:

happyString.each do |ss|  
if ss == "sad"  
ss = "Happy"  
elsif ss == "Sad"  
ss = "Happy"  
end

But then I realised that I was using the wrong method, .collect! does the trick though:

def add_more_ruby(string)

    happyString = string.split(" ")

    happyString.collect! { |element| (element == "sad") ? "happy" : element}
    happyString.collect! { |element| (element == "Sad") ? "Happy" : element}

    doneString = happyString.join(" ")

    puts doneString


end

add_more_ruby("The clowns were sad.")

.collect! works like this:

array.collect! {|variable| block } -> a new array that has been processed through block. In the above the block is (element == "sad") ? "happy" : element}. i.e. it asks if the array element is equal to sad, and if it is, it is changed to happy.

At this point i started to cry real crocodile tears because even after all this odyssey the happy string did not do something magical and clever with period. Hence, back to .gsub:

string.gsub("sad", "happy").gsub("Sad", "Happy")

Rake loved it.

1.6 More string glory

My solution (with sage advice from Omar): 

def odds_and_evens(string, return_odds)  
    result = []
    if return_odds
        string.split("").each_with_index { |v, i| result << v if i.odd?}
    else
        string.split("").each_with_index { |v, i| result << v if i.even?}
    end

    result.join("") 
end

i.e.

  1. Define function
  2. Create empty array (called results)
  3. If return odd is true we will split the string into an array, then we will do something nice to each element and add an index
  4. We add the value (v) to the empty array called results if index (i) is even
  5. Then we finish by joining the array, as this is the last thing in the method, this is what it will return.

Makers Academy's solution is much shorter:

def odds_and_evens(string, return_odds)  
  to_return = ""
  string.size.times do |index|
    next if return_odds && index.even?
    next if !return_odds && index.odd?
    to_return << string[index]
  end
  to_return
end
  1. Define function
  2. Create an empty string
  3. (HAWT!) string.size.times checks how many characters it has and then performs something cool that amount of times
  4. Do |index|

1.7 Pesky nills in my string

Again, I thought this was pretty cool:

def pirates_say_arrrrrrrrr(string)  
    piratespeak = ""

    string.split("").each_with_index {|v, i| piratespeak << string[i+1] if v == "r" || v == "R"}            

    return piratespeak
end

Rake did not agree with me. The problem: when you feed it a string where r is at the end, it tries to include nil in the string - giving me this rake error: " no implicit conversion of nil into String".

How then to fix it? Maybe we could do something adding another precondition that i+1 must be smaller than string size?

def pirates_say_arrrrrrrrr(string)  
    piratespeak = ""

    string.split("").each_with_index {|v, i| piratespeak << string[i+1] if  i+1 < string.size && v == "r" || v == "R" }         

    return piratespeak
end

Yes! it worked. But! its not as cool as:

def pirates_say_arrrrrrrrr(string)  
  to_return = ""
  add_next = false
  string.size.times do |index|
    current_char = string[index,1] # the second param here tells it to get a str of length 1.

    to_return << current_char if add_next
    add_next = (current_char == "r" || current_char == "R")
  end
  to_return
end

Here, the makers masters do the following awesomness:

  1. Define the function
  2. Create a new empty string
  3. Define the add_next variable and set it to false
  4. Create a iterator dealing with index that will run for the length of the string
    4a. Define current character as a string, one in length
    4b. Add the current character to the empty string to be returned after loop IF add next is true
    4c. Then! The geniusness comes into our lives, and boy, what genius it is. As this is a loop they now set check if the current character is r or R. If it is the next time the loop is run and we are back on step 4b, it will add it if the last character was r.
  5. Then they just return the now filled string.

Voila! What glory!

However, I am not sure that is better than mine? Turns out it is: runs 0.00002 seconds faster. Bet it is because I check to make sure i + 1 is not longer than the full string each time a character is checked.
